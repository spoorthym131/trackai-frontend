# trackai-frontend

React + TypeScript frontend for TrackAI — a job application tracker with AI-powered JD-to-resume matching.

## Why I'm building this

As an MS CS candidate entering a high-volume job search, I needed a way to track 100+ applications across companies, sources, and pipeline stages — and a way to quickly evaluate how well each role fits my background before I apply. Rather than use a spreadsheet, I'm building the tool I actually need.

## What the UI does

The dashboard opens on a Kanban board showing applications grouped by stage, with a toggle to a sortable table view for strategic review. Users add applications through a form, drag cards to update status, and see aggregate stats (response rate, conversion by stage, breakdown by source). The JD matcher lives on the application detail page — paste a JD, hit analyze, and see a skill-match table, overall fit score, and top gaps, all saved alongside the application.

## Tech stack

- **React + TypeScript + Vite** — UI framework and build tool
- **Tailwind CSS + shadcn/ui** — styling and component library
- **React Router** — client-side routing
- **TanStack Query** — server state management
- **Deployed on Vercel**
